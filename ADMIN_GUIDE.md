## 1. Вход в админку

Откройте в браузере:

```
http://<адрес-сервера>/admin
```

## 2. Обзор разделов

В левом меню расположены шесть разделов:

| Раздел | Что делает |
|---|---|
| Глобальные настройки | Промт для GigaChat и текст в конце PDF |
| Сценарии | Список тем диагностики, которые видит пользователь |
| Шаги сценариев | Вопросы внутри каждой темы |
| Настройки валидации | Правила проверки качества ответов |
| Сессии диалогов | История всех разговоров (только просмотр) |
| Отчёты | Сгенерированные PDF-отчёты (только просмотр) |

## 3. Сценарии

Сценарий — это тема диагностики, которую пользователь выбирает в самом начале диалога (например, «Падение продаж» или «Проблемы с персоналом»).

### Создание сценария

1. Перейдите в **Сценарии** → нажмите **Create**.
2. Заполните поля:

   - **Название** — отображается пользователю в списке выбора. Формулируйте коротко и понятно.
   - **Описание** — внутренняя заметка, пользователь не видит.
   - **Активен** — если галочка снята, сценарий скрыт из чата. Используйте для временного отключения без удаления.
   - **Дополнительный промт сценария** — уточнения для GigaChat, специфичные для этой темы (подробнее в разделе 5).

3. Нажмите **Save**.

> После создания сценария добавьте к нему вопросы в разделе **Шаги сценариев**.

### Редактирование и удаление

- Кликните по строке сценария → **Edit**.
- Удаление сценария автоматически удаляет все его шаги.

## 4. Шаги сценариев (вопросы)

Шаг — это один вопрос, который бот задаёт пользователю. Вопросы задаются строго по порядку.

### Добавление вопросов

1. Перейдите в **Шаги сценариев** → нажмите **Create**.
2. Заполните поля:

   - **Scenario** — выберите сценарий из списка.
   - **Order index** — порядковый номер вопроса, начиная с `1`. Вопросы задаются от меньшего к большему.
   - **Message text** — текст вопроса, который увидит пользователь.

3. Нажмите **Save**.

### Важные правила

- Нумерация должна быть **без пропусков**: 1, 2, 3... Если создать шаги 1, 2, 4 — третий вопрос никогда не будет задан.
- Два шага с одинаковым `order_index` в одном сценарии **недопустимы** — система вернёт ошибку.
- Пользователь видит вопросы в формате **«Вопрос 2/7»** — общее количество считается автоматически.

### Изменение порядка вопросов

Чтобы поменять вопросы местами, отредактируйте `order_index` у каждого из них вручную.

## 5. Глобальные настройки

Это единственная запись-синглтон — создать новую или удалить её нельзя, только редактировать.

### Универсальный промт GigaChat

Базовая инструкция для ИИ, которая применяется ко **всем** сценариям. Именно здесь задаётся структура отчёта.

**Как работают разделы отчёта:**

PDF формирует ровно те разделы, которые вернёт GigaChat. Каждый раздел должен начинаться с заголовка первого уровня:

```
# ОСНОВНАЯ ПРОБЛЕМА
# УРОВЕНЬ ЦИФРОВОЙ ЗРЕЛОСТИ
# ТЕКУЩЕЕ СОСТОЯНИЕ
# РЕКОМЕНДАЦИИ
```

Чтобы добавить новый раздел в отчёт — добавьте его заголовок в промт с описанием, что туда писать.

**Связь с промтом сценария:**

Если у сценария задан дополнительный промт, он **добавляется** к глобальному через разделитель. Дополнительный промт сценария может переопределить список разделов или добавить уточняющие инструкции для конкретной темы.

### Текст блока «Следующий шаг» в PDF

Фиксированный текст, который печатается в конце каждого отчёта в выделенном блоке. Обычно содержит призыв к действию или контакты. Поддерживает маркированные списки через `- пункт`.

## 6. Настройки валидации ответов

Система автоматически проверяет качество каждого ответа пользователя через GigaChat и относит его к одному из трёх типов:

| Тип | Описание | Пример |
|---|---|---|
| ТИП 1 — Неосмысленный | Случайные символы, ответ полностью не по теме, намеренный отказ | «аааа», «пропустить», «неважно» |
| ТИП 2 — Слабый | Пользователь не владеет темой, ответ очень короткий и без содержания | «не знаю», «нормально», «всё плохо» |
| ТИП 3 — Осмысленный | Описывает реальную ситуацию, даже если коротко | «CRM не используем», «всё в Excel» |

Настройки — синглтон (только редактирование).

### Параметры

**Валидация включена** — главный переключатель. Если выключить, все ответы принимаются без проверки.

---

**Неосмысленный ответ — повторять вопрос (Тип 1)**

Если включено: бот не переходит к следующему вопросу, а отправляет сообщение с просьбой ответить заново.

- **Сообщение при неосмысленном ответе** — текст, который увидит пользователь. Вставьте `{question}` в то место, где должен появиться повторный вопрос:

  ```
  Я не смог понять ваш ответ. Пожалуйста, ответьте подробнее.

  {question}
  ```

---

**Слабый ответ — отмечать в отчёте (Тип 2)**

Если включено: бот принимает ответ и переходит дальше, но добавляет к нему невидимую для пользователя пометку. GigaChat видит эту пометку при генерации отчёта и учитывает низкое качество ответа.

- **Пометка к слабому ответу для ИИ** — строка, добавляемая к ответу. Например:
  ```
  [Уровень ответа: низкий — пользователь не владеет темой]
  ```

---

**Промт для классификации ответов**

Инструкция для ИИ, который определяет тип ответа. Содержит два обязательных плейсхолдера — не удаляйте их:

- `{question}` — текст вопроса бота
- `{answer}` — ответ пользователя

## 7. Сессии диалогов

Раздел только для просмотра — создание и редактирование недоступны.

Здесь отображаются все диалоги пользователей. Статусы сессии:

| Статус | Значение |
|---|---|
| `pending_scenario` | Пользователь ещё не выбрал тему |
| `in_progress` | Идут вопросы сценария |
| `pending_contact` | Бот собирает контактные данные |
| `generating` | Идёт генерация отчёта |
| `completed` | Отчёт сгенерирован и готов |

---

## 8. Отчёты

Раздел только для просмотра — создание и редактирование недоступны.

Здесь хранятся все сгенерированные PDF-отчёты. Для каждого отчёта видно:

- **Session** — ID сессии, к которой относится отчёт
- **Created at** — время генерации
- **Sent to Bitrix** — был ли отчёт передан в Bitrix24
- **Bitrix deal ID** — номер сделки в Bitrix24 (если передача прошла успешно)
